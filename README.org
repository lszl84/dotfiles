#+TITLE: Dotfiles
#+DESCRIPTION: Configuration files for Linux (dwm / X11)

* Overview

Personal dotfiles for a Linux desktop built around dwm on X11. Everything is
managed with GNU Stow — each top-level directory is a stow package that
symlinks into ~$HOME~.

* Contents

** dwm

Tiling window manager (dwm 6.5) with a complete desktop environment.

~dwm-build~ downloads dwm and st from suckless.org, applies patches via Python
scripts, copies in ~dwm-config.h~ / ~st-config.h~, and installs binaries to
=$HOME/.local/bin/=.

Applied dwm patches:
- *status2d* — colored status bar segments (~^c#RRGGBB^~)
- *lock* — built-in PAM screen locker (fullscreen black window, keyboard grab, color feedback)
- *vtblack* — black overlay on VT switch to prevent flash

Applied st patches:
- *scrollback* — 2000-line history with Shift+PageUp/Down
- *scrollback-mouse* — Shift+scroll wheel
- *anysize* — dynamic border centering for arbitrary window sizes

Key features:
- HiDPI (192 DPI / 2x) scaling for 3120x2080 display
- st terminal with Modus Vivendi colors
- rofi launcher, PipeWire volume keys, brightnessctl
- Picom compositor: vsync, 35% inactive dimming
- Built-in PAM screen locker triggered via ~xss-lock~ (no i3lock)
- Idle lock at 20 min, suspend 30s after lock
- Status bar: volume, battery (with color + low battery notification), uptime (red warning after 14 days), clock
- Polkit agent (polkit-gnome) for GUI privilege prompts
- GTK 3/4 dark theme via xsettingsd + gsettings (xsettingsd REQUIRED for HiDPI GTK scaling)
- Dunst notifications (dark theme, top-right, rounded corners)

See [[dwm/.config/dwm/DWM.org][DWM.org]] for full documentation.

** bash

Shell configuration:
- Colored output for ~ls~ and ~grep~
- Minimal prompt showing current directory
- PATH additions for =$HOME/.local/bin=
- ~ffgrab~ alias for hardware-accelerated screen recording (VAAPI)
- ~cco~ alias for ~claude --continue~

** emacs-easy

A clean Emacs setup:
- CUA keybindings (Ctrl+C/X/V/Z work as expected)
- Modus Vivendi dark theme with ~#000000~ background
- Custom modeline: rounded pill indicator with modified dot, buffer/mode/position
- Magit with quick-push (~C-c p~)
- ~C-c r~ — cmake build and run (~cmake --build build && ./build/main~)
- Auto-close compilation buffer on success
- Org mode with clock persistence
- Emacs server for fast ~emacsclient~ usage

** touchegg

Touchpad gesture daemon for three-finger swipe workspace switching:
- Swipe left → next tag (natural direction)
- Swipe right → previous tag

Uses ~SEND_KEYS~ (~Ctrl+Alt+Left/Right~) rather than ~CHANGE_DESKTOP~ for
reliability.

** fastfetch

System info display with compact logo: OS, kernel, uptime, packages, shell, DE,
CPU, GPU, memory, swap, disk, local IP, battery.

** ffmpeg

Audio/video utility scripts:
- ~ffnorm.sh~ — loudness normalization with limiter, EQ, and LUFS measurement
- ~ffrecord.sh~ — auto-incrementing WAV recording from PulseAudio
- ~ffstreamsilent.sh~ — VAAPI-accelerated screen streaming to RTMP (silent audio)
- ~ffgrab~ (bash alias) — VAAPI-accelerated screen + audio recording

** gdb

GDB config enabling ~debuginfod~ for automatic debug symbol downloads.

* Installation

** Dependencies (Arch Linux)

#+begin_src bash
# Core
sudo pacman -S xorg xorg-xinit lightdm lightdm-slick-greeter stow git

# dwm build deps
sudo pacman -S base-devel libx11 libxft libxinerama python

# Desktop (xsettingsd is REQUIRED for GTK HiDPI scaling - do not skip!)
sudo pacman -S picom rofi xss-lock brightnessctl polkit-gnome xorg-xsetroot xsettingsd gsettings-desktop-schemas dunst

# Desktop portal (required for Firefox/libadwaita to detect dark mode)
sudo pacman -S xdg-desktop-portal xdg-desktop-portal-gtk

# Audio
sudo pacman -S pipewire pipewire-pulse wireplumber

# Fonts (bitmap for TTY, vector for X11)
sudo pacman -S adwaita-fonts ttf-jetbrains-mono-nerd terminus-font

# Touchpad gestures
sudo pacman -S touchegg

# Emacs
sudo pacman -S emacs

# Hardware video encoding (VAAPI — for ffgrab/ffstreamsilent)
sudo pacman -S intel-media-driver libva-intel-driver

# Optional
sudo pacman -S fastfetch ffmpeg gdb
#+end_src

** Setup

#+begin_src bash
cd ~/dotfiles

# Symlink configs
stow bash
stow dunst
stow dwm
stow emacs-easy
stow touchegg
stow fastfetch
stow gdb
stow ffmpeg

# Build and install dwm + st
dwm-build

# Enable touchegg daemon
sudo systemctl enable --now touchegg

# Enable LightDM
sudo systemctl enable lightdm
#+end_src

*** Desktop Portal (Firefox dark mode)

Firefox and other libadwaita apps use XDG Desktop Portal to detect the system dark mode. Install the packages above, then add to your =~/.xinitrc= before starting dwm:

#+begin_src bash
# Start desktop portal for Firefox/libadwaita dark mode detection
/usr/lib/xdg-desktop-portal-gtk &
/usr/lib/xdg-desktop-portal &
#+end_src

*** GNOME Keyring (Password/Key Storage)

For secure password and SSH key storage with automatic unlock at login:

#+begin_src bash
# Install keyring daemon and GUI manager
sudo pacman -S gnome-keyring seahorse
#+end_src

**** PAM Setup (Auto-unlock with login password)

Add to PAM configuration to automatically unlock keyring when you log in:

#+begin_src bash
# Edit /etc/pam.d/login
sudo nano /etc/pam.d/login
#+end_src

Add these lines to the file:
#+begin_src
auth       optional     pam_gnome_keyring.so
session    optional     pam_gnome_keyring.so auto_start
#+end_src

**** Start Keyring in .xinitrc (REQUIRED for startx users)

The PAM configuration above only works with display managers (LightDM, GDM, etc.).
If you use ~startx~ (like this dwm setup), you MUST add this to your =~/.xinitrc=:

#+begin_src bash
# Start GNOME Keyring daemon (REQUIRED for secret-tool and seahorse with startx)
eval $(gnome-keyring-daemon --start --components=pkcs11,secrets,ssh)
export SSH_AUTH_SOCK
#+end_src

This is already included in the dwm ~.xinitrc= provided in this repo.

**** Using Seahorse GUI

Launch the graphical keyring manager to browse and manage stored passwords:

#+begin_src bash
seahorse
#+end_src

Or use command line:
#+begin_src bash
# Store a password
secret-tool store --label="My Password" service myapp username me

# Retrieve a password
secret-tool lookup service myapp username me
#+end_src

The ~dwm-build~ script handles everything: downloading sources, applying
patches, compiling, installing to =$HOME/.local/bin/=, and creating the
=/etc/pam.d/dwm= config for the built-in locker. After a reboot, select "dwm"
from the LightDM session menu.

** Console Font (TTY / Boot)

Terminus font for high-DPI displays in virtual consoles (TTYs) and boot:

#+begin_src bash
# Install terminus font (already included in Fonts above)
sudo pacman -S terminus-font

# Set large font for current TTY (ter-132n = 16x32, good for HiDPI)
setfont ter-132n

# Make persistent across boot and all TTYs
sudo tee /etc/vconsole.conf << 'EOF'
FONT=ter-132n
EOF

# Regenerate initramfs to include font for LUKS password prompt
sudo mkinitcpio -P
#+end_src

Available sizes: ~ter-116n~ (11x22), ~ter-124n~ (12x24), ~ter-132n~ (16x32), ~ter-140n~ (16x40). Run ~showconsolefont~ to preview.

Note: TTY font takes effect immediately on next VT switch or boot. The ~vconsole.conf~ is read by ~systemd-vconsole-setup~ during boot. ~mkinitcpio -P~ rebuilds initramfs so the font is available at the LUKS password prompt.

** Disk Migration (dd to new drive)

To migrate your entire system to a larger/faster disk (e.g., upgrading to 4TB Samsung 990 EVO):

*** Prerequisites

- Boot from Arch Live USB
- Connect both old and new drives
- Know device names: ~lsblk~ to identify (e.g., ~/dev/nvme0n1~ = old, ~/dev/nvme1n1~ = new)

*** Clone Process

#+begin_src bash
# 1. Open the LUKS container on the old disk
sudo cryptsetup open /dev/nvme0n1p2 cryptold

# 2. Clone the entire disk (preserves LUKS, LVM, everything)
# Whole disk clone (if new disk is same or larger size):
sudo dd if=/dev/nvme0n1 of=/dev/nvme1n1 bs=4M status=progress conv=fsync

# OR clone just the LUKS partition:
# sudo dd if=/dev/mapper/cryptold of=/dev/nvme1n1p1 bs=4M status=progress conv=fsync
#+end_src

*** Post-Clone Expansion

After booting from the new disk:

#+begin_src bash
# 1. Open LUKS on the NEW disk
sudo cryptsetup open /dev/nvme1n1p2 cryptlvm

# 2. Resize physical volume to use full disk space
sudo pvresize /dev/mapper/cryptlvm

# 3. Extend logical volumes to fill free space
# Option A: Extend home only
sudo lvextend -r -l +100%FREE /dev/ArchinstallVg/home

# Option B: Split between root and home
# sudo lvextend -r -L +100G /dev/ArchinstallVg/root
# sudo lvextend -r -l +100%FREE /dev/ArchinstallVg/home
#+end_src

*** IMPORTANT: Resize LUKS Partition (for larger disks)

If you cloned to a disk larger than the original (e.g., 4TB), the LUKS partition will still be the original size (~476GB). You must resize the physical partition first:

**** From Arch Live USB:

#+begin_src bash
# 1. Boot from Arch Live USB
# 2. Open the LUKS container
sudo cryptsetup open /dev/nvme0n1p2 cryptlvm

# 3. Resize the physical partition to fill the disk
# Using parted (non-interactive):
sudo parted /dev/nvme0n1 resizepart 2 100%

# Or using gparted (GUI - easier):
# sudo gparted /dev/nvme0n1
# Right-click partition → Resize/Move → Drag to fill disk → Apply

# 4. Resize LUKS to match new partition size
sudo cryptsetup resize cryptlvm

# 5. Resize LVM physical volume
sudo pvresize /dev/mapper/cryptlvm

# 6. Now extend home to use all free space (~3.5TB on 4TB disk)
sudo lvextend -r -l +100%FREE /dev/ArchinstallVg/home

# 7. Verify the resize
lsblk
# Should show: cryptlvm = 3.6T, ArchinstallVg-home = ~3.5T
#+end_src

**** Common Issue:

If you only run ~lvextend~ without resizing the partition first:
- ~lvextend~ reports "New size matches existing size"
- ~lsblk~ shows cryptlvm still at ~476GB
- Solution: You skipped step 3 (resize partition). Boot Live USB and follow steps above.

*** Bootloader Setup

If whole disk was cloned, the bootloader should work. If just the LUKS partition was cloned:

#+begin_src bash
# Mount the new system
sudo mount /dev/ArchinstallVg/root /mnt
sudo mount /dev/nvme1n1p1 /mnt/boot

# Reinstall systemd-boot
sudo arch-chroot /mnt bootctl install
#+end_src

*** Update fstab/crypttab (if needed)

If disk UUIDs changed:

#+begin_src bash
# Get new UUIDs
lsblk -f

# Update /etc/crypttab with new LUKS UUID
sudo nano /etc/crypttab

# Update /etc/fstab if any UUIDs changed
sudo nano /etc/fstab

# Regenerate initramfs
sudo mkinitcpio -P
#+end_src

*** Notes

- ~dd~ copies block-level: LUKS, LVM, ext4, all preserved exactly
- The ~-r~ flag in ~lvextend~ auto-resizes filesystems while mounted
- Can resize root (/) and home live while booted from the new disk

** Alternative Desktop Environment (XFCE)

For screencasts or when you need a full desktop environment with modern window decorations and proper HiDPI:

*** Install XFCE

#+begin_src bash
sudo pacman -S xfce4 xfce4-goodies xfce4-terminal
#+end_src

*** Configure HiDPI for XFCE

After starting XFCE, open Settings → Appearance and set:

- Window Scaling: 2x (HiDPI)
- Default font: JetBrains Mono Nerd Font, 12pt
- Icon theme: Adwaita

Then open Settings → Window Manager and set:
- Style: Default (or any modern theme)
- Button layout: Title, Minimize, Maximize, Close

*** Switch to XFCE

#+begin_src bash
# Toggle from dwm to XFCE
switch-wm xfce

# Or toggle back to dwm
switch-wm dwm

# Then restart X session
startx
#+end_src

*** Using switch-wm script

The ~switch-wm~ script is installed to =~/.local/bin/~ when you stow bash:

- ~switch-wm~ — toggles between dwm and XFCE automatically
- ~switch-wm dwm~ — switch to dwm explicitly
- ~switch-wm xfce~ — switch to XFCE explicitly

*** XFCE vs dwm

| Feature | dwm | XFCE |
|---------|-----|------|
| Tiling | Dynamic | Manual (tiling plugin available) |
| Title bars | None | Yes (modern, proper HiDPI) |
| Look | Minimal | Traditional desktop |
| HiDPI | Via Xft/XSETTINGS | Native 2x scaling support |
| Resource use | Very low | Low-moderate |

*** Tips

- Use dwm for daily work (efficiency)
- Use XFCE for screencasts (familiar look, window decorations)
- Both use your existing: dunst, picom, HiDPI settings
- XFCE has proper window manager with 2x scaled decorations
