#+TITLE: Dotfiles
#+DESCRIPTION: Labwc Wayland desktop on Arch Linux

* Overview

Personal dotfiles for a minimal [[https://labwc.github.io/][labwc]] desktop on Arch Linux.
Labwc is a Wayland stacking compositor inspired by Openbox -- lightweight,
fast, and simple.

Managed with [[https://www.gnu.org/software/stow/][GNU Stow]]. Each top-level directory is a stow package that
symlinks into =$HOME=.

The status bar is [[https://github.com/lszl84/simpbar][simpbar]] -- a lightweight Wayland bar written in pure C
with Cairo rendering, HiDPI support, and workspace indicators via the
=ext-workspace-v1= protocol.

| Package      | What it configures                              |
|--------------+-------------------------------------------------|
| =labwc=      | Compositor, keybinds, autostart, kanshi, theme  |
| =foot=       | Terminal emulator                               |
| =fuzzel=     | Application launcher                            |
| =dunst=      | Notification daemon                             |
| =waybar=     | Status bar configs + shared style (for hyprland)|
| =wallpapers= | Desktop wallpapers                              |
| =bash=       | Shell config, prompt, aliases                   |
| =emacs-easy= | Emacs with CUA keys, Modus Vivendi theme        |
| =scripts=    | Utility scripts in =~/.local/bin=               |
| =fastfetch=  | System info display                             |
| =gdb=        | GDB with debuginfod                             |
| =hyprland=   | Hyprland compositor (alternative, not required) |
| =wireplumber= | Per-sink speaker EQ/HPF (Huawei MateBook X)    |

* Installation on Fresh Arch Linux

** 1. Install packages

#+begin_src bash
# Compositor and session
sudo pacman -S labwc kanshi swaybg swaylock swayidle wlopm

# Terminal and launcher
sudo pacman -S foot fuzzel

# Status bar -- build and install simpbar from https://github.com/lszl84/simpbar
# Or use waybar: sudo pacman -S waybar

# Notifications and screenshots
sudo pacman -S dunst grim slurp

# Audio and brightness
sudo pacman -S pipewire wireplumber brightnessctl

# Fonts
sudo pacman -S ttf-jetbrains-mono-nerd adwaita-fonts noto-fonts

# GTK dark theme
sudo pacman -S gnome-themes-extra

# Stow and git
sudo pacman -S stow git

# Clipboard
sudo pacman -S wl-clipboard

# Emacs (optional)
sudo pacman -S emacs
#+end_src

** 2. Clone and stow

#+begin_src bash
cd ~
git clone <your-repo-url> dotfiles
cd dotfiles

# Core desktop
stow labwc foot fuzzel dunst wallpapers bash

# If using waybar (hyprland or as fallback for labwc)
stow waybar

# Optional
stow emacs-easy
stow fastfetch
stow gdb
stow scripts
#+end_src

** 3. Start labwc

From a TTY (Ctrl+Alt+F2), log in and run:

#+begin_src bash
exec labwc
#+end_src

Or add to =~/.bash_profile= to auto-start on login:

#+begin_src bash
if [ -z "$WAYLAND_DISPLAY" ] && [ "$(tty)" = "/dev/tty1" ]; then
    exec labwc
fi
#+end_src

* What You Get

** Desktop

- Zero gaps, zero borders -- every pixel is usable space
- Bottom bar ([[https://github.com/lszl84/simpbar][simpbar]]) with workspace indicators, volume, battery, and clock
- HiDPI window buttons (close only) from the GTK theme
- Drop shadows on floating windows
- Natural scrolling on touchpad, two-finger click for right-click
- HiDPI scaling at 1.75x via kanshi (2880x1800 laptop display)
- Dark theme (Adwaita-dark) with subtle blue terminal background
- GNU wildebeest wallpaper

* Customization

** Adding a monitor

Edit =~/.config/kanshi/config=:

#+begin_src
profile docked {
    output eDP-1 mode 2880x1800@120Hz position 0,0 scale 1.75
    output DP-2 mode 3840x2560@60Hz position 1646,0 scale 1.6
}
#+end_src

** Changing wallpaper

Wallpapers live in =~/.local/share/wallpapers/=. Change the filename
in =~/.config/labwc/autostart=:

#+begin_src bash
swaybg -i ~/.local/share/wallpapers/your-wallpaper.jpg &
#+end_src

** Changing workspace count

Edit =~/.config/labwc/rc.xml=, find the =<desktops>= section:

#+begin_src xml
<desktops>
  <popupTime>0</popupTime>
  <names>
    <name>1</name>
    <name>2</name>
    <name>3</name>
    <name>4</name>
  </names>
</desktops>
#+end_src

* GNOME: replicate this setup on a new machine

The =gnome/= directory ships a single script that installs and configures the
extension stack used here (Dash to Panel, ArcMenu, Tiling Shell) plus
=dnf5-plugin-automatic= for security updates.

#+begin_src bash
git clone <this-repo> ~/dotfiles
~/dotfiles/gnome/install.sh
# log out, log back in
#+end_src

The script:
- installs =gnome-extensions-cli= via =pipx=
- installs the three extensions from extensions.gnome.org
- detects the target monitor ID via Mutter's DBus (vendor + serial), substitutes
  it into the Dash to Panel config (the source machine's ID was =VXN-0x19c5fee7=
  — different on every laptop)
- loads dconf snapshots for all three extensions
- configures =dnf5-automatic.timer= for security-only updates

Refresh the snapshots before committing if you've tweaked anything:

#+begin_src bash
cd ~/dotfiles/gnome
dconf dump /org/gnome/shell/extensions/dash-to-panel/ | \
  sed 's/VXN-0x19c5fee7/__MONITOR_ID__/g' > dash-to-panel.dconf
dconf dump /org/gnome/shell/extensions/arcmenu/        > arcmenu.dconf
dconf dump /org/gnome/shell/extensions/tilingshell/    > tilingshell.dconf
#+end_src

(Use your own monitor ID in the =sed= — find it with the same =busctl= snippet
the install script uses.)

* Fedora: automatic security updates

On Fedora 43+, the =dnf5-plugin-automatic= package (already part of the base
system) can apply security updates unattended. Combined with an hourly
=dnf needs-restarting -r= check (planned for simpbar), this gives a hands-off
update flow with a visible reboot indicator.

** Setup

The plugin ships a config template; copy it into place, then flip two values:

#+begin_src bash
sudo install -m 0644 /usr/share/dnf5/dnf5-plugins/automatic.conf /etc/dnf/automatic.conf

sudo sed -i \
  -e 's/^apply_updates = .*/apply_updates = yes/' \
  -e 's/^upgrade_type = .*/upgrade_type = security/' \
  /etc/dnf/automatic.conf
#+end_src

Defaults that are already correct: =download_updates = yes=, =emit_via = stdio=,
=reboot = never= (we surface the reboot need via simpbar instead).

Enable the daily timer (note the =5= — the legacy =dnf-automatic.timer= is a
compat shim, prefer the native one):

#+begin_src bash
sudo systemctl enable --now dnf5-automatic.timer
#+end_src

** Verifying

#+begin_src bash
systemctl list-timers dnf5-automatic.timer
sudo systemctl start dnf5-automatic.service   # trigger one run now
journalctl -u dnf5-automatic.service --since '1 day ago'
dnf needs-restarting -r                       # exit 1 = reboot needed
#+end_src

** Inspecting what was installed

The =stdio= emitter sends output to journald, and every transaction is also
recorded in =dnf history= and =/var/log/dnf5.log=:

#+begin_src bash
# All runs of the automatic service
journalctl -u dnf5-automatic.service

# This boot only
journalctl -u dnf5-automatic.service -b 0

# When the timer fired (regardless of output)
journalctl -u dnf5-automatic.timer

# All transactions, manual + automatic
sudo dnf history list
sudo dnf history info <id>

# Raw rpm/dnf log
less /var/log/dnf5.log
#+end_src

Non-security updates are still applied manually with =sudo dnf upgrade=.
